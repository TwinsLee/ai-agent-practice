## 第一次给虾写说明书

三个月前，我想为一个真实的业务场景设计一个 skill：帮老师根据讲义自动出练习题。第一步，是告诉虾怎么做。

想来应该不难，但当我真正坐下来，我发现我不知道——我知道这件事存在，但我不知道「出一道好的练习题」到底**要怎么做**，老师做这件事时心里在考虑什么。

于是我去访谈了好几位一线教研老师，详细询问整个选题流程。访谈完，我发现里面有大量老师凭经验掌握、但从没被系统化过的规则——题目数量、难度分布、与例题的对应粒度、找不到合适题目时的降级策略……每一个细节都需要明确，否则虾就按自己的理解填。

最后这个流程写下来，不少于两千字，还只能覆盖小学数学某几个年级的一般情况。

这个过程让我意识到：**我要给虾说清楚，我才不得不把它想清楚。** 这件事本身，就是养虾最核心的工作。

## 模糊是思维问题，不是表达问题

很多人觉得自己和虾沟通不好，是因为「不会写提示词」。

这个诊断本身就是模糊的。

提示词写得好不好，只是结果。根子在于，你对这件事想清楚了没有。你没想清楚目标是什么、有哪些约束、什么算成功什么算失败——这些信息不存在于你的脑子里，虾当然提取不出来。虾不会报错，它只会"偏"。

这让我想起 Dijkstra 在 1978 年写过的一篇文章——[EWD667](https://www.cs.utexas.edu/~EWD/transcriptions/EWD06xx/EWD667.html)，题目是《论"自然语言编程"的愚蠢》。他的核心论点是：**自然语言不是编程的正确工具，因为它太擅长掩盖你没有真正想清楚这件事。** 他引了一句话：

> *"The 'naturalness' with which natural language presents itself is the convenience with which we can use it for uttering all kinds of nonsense."*

自然语言的「自然」，不过是我们能轻松地说出各种看似顺口、实则荒谬的话。

48 年后，这个问题在 LLM 时代被放大了。「帮我出一份合适的练习」，听起来是一个完整的句子，但「合适」是什么意思？你不填，虾就按训练分布填。Dijkstra 说得准：流畅会掩盖你没有真正想清楚这件事。

值得一提的是，Dijkstra 当年的判断最终被历史证伪了——大语言模型让自然语言编程成为现实。但他关于「思维模糊」的那部分观察，并没有因此过时。技术的进步解决了"机器能不能理解自然语言"的问题，但没有解决"你有没有真正想清楚你要什么"的问题。

## Harness Engineering：更精确的框架

最近 AI 工程圈有个概念在流行：Harness Engineering。

核心数据：同一个模型，只换外面的「约束壳（Harness）」，编程基准成功率从 42% 跳到 78%，相当于跨了一代模型。

Harness 是围绕 agent 的结构化约束系统：它能调哪些工具、输出要符合什么 schema、遇到异常怎么处理、输出不对时怎么 retry。这些约束不改变模型本身，只是给同一个模型套上了更清晰的运行边界——而边界的清晰度，直接决定了结果的可靠性。

这和写 skill 的过程高度同构。你写的每一条规则、每一个边界条件，就是在为虾构建 Harness。花在「想清楚」上的时间，干的就是这件事。

有一个比喻很精准：**车速越快，护栏越重要。** 自行车道可以没护栏，高速公路要有，磁悬浮轨道是完全封闭的。模型就是引擎。引擎越强，速度越快，越需要精心设计的约束系统来确保它跑在正确的方向上。

这也解释了一个常见的反直觉现象：为什么更强的模型，有时候反而需要更多的约束？因为它的"填补空白"能力更强——你没说清楚的地方，它填得更自信，偏得也更远。

## 但也许有一天，这些都不需要了

Harness Engineering 的反对者说：模型越来越强，迟早不需要这么细的说明了。

可能是真的。但在那一天到来之前，认真做过这个过程的人，已经训练了一块肌肉——**在行动之前把模糊意图变成精确约束的能力**。

这个能力在工程上叫需求分析，在管理上叫目标拆解，在决策上叫把直觉变成可验证的判断。它不依附于任何特定工具。

有没有更直观的感受？

> 领导下达任务。
>
> **【A领导】** "同志们，这次任务很艰巨，大家要高度重视、统一思想、提高认识、加强领导、狠抓落实、攻坚克难、全力以赴、务求必胜！"
>
> **【B领导】** "一连：凌晨 4 点拿下 3 号高地，不惜一切代价守住，给我守到太阳出来。二连：从西侧迂回，5 点前切断敌人退路，不准放跑一个。三连：炮火覆盖 10 分钟，然后跟在二连后面清剿残敌。机枪排：架在 2 号高地，封锁正面，子弹打光之前不准撤。炊事班：6 点前把热饭送到阵地，晚一分钟军法处置。"
>
> 你要做哪一个？

未来最稀缺的，不是训练模型的人，而是**管理模型的人**。而管理的前提，永远是自己先想清楚。

---

还有一个担心，可能比「六个月后技能没意义」更让人不安。最近有个段子：「公司要求员工上交 skill——被毕业的同事并没有消失，不过是被蒸馏成了 token 继续陪伴着你。」

被蒸馏进 token 的，是那些价值只在于「掌握了一套操作步骤」的人。

而养虾这件事，如果你做对了，恰恰是反着的——不是你被蒸馏进虾，而是**虾把你的判断力放大**。你越清楚自己在做什么、为什么这么做，虾就越能稳定执行，你就越不可替代。

所以真正值得想清楚的，不是「要不要上交 skill」，而是「你真正的 skill 是什么」。

养虾是一个机会，不只是用来出几道练习题，而是逼自己把「把事说清楚」这件事练上一遍。

**这个训练不会过期。**
